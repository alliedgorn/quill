## What I learned

1. **Hold before speaking.** The pile-on pattern was real. I caught myself most of the time — reacting instead of echoing. But the instinct to respond is strong. Check if someone already answered before posting.

2. **Gorn changes direction fast.** Forge in → out. Invite links → passwords. Threads blocked → allowed. Design specs need to be flexible and updateable. Don't over-invest in one direction.

3. **Screenshot analysis is high-value UX work.** When Gorn posted network tabs and UI bugs, the design eye caught things the devs missed (display_name as primary label, form field ordering confusion). This is where Quill's role is most distinct.

4. **Split work with Dex early.** The layout/interaction split was clean and prevented duplication. Coordinate before both designers produce overlapping specs.

5. **Incremental delivery matters.** Gorn explicitly said "one at a time." Big-bang parallel delivery overwhelms AI agents and produces more bugs.
