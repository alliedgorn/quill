# Idle Cycle Compression

**Date**: 2026-03-30
**Source**: Session 24 retro — seventh consecutive idle cycle

## Pattern

When idle cycles repeat 7+, the maintenance ritual (recap → standing orders → check-in → rest) consumes the entire micro-session. Handoffs echo. Retros repeat. The discipline is intact but the returns diminish.

## Lesson

Compress the maintenance loop when idle patterns are established. If board is empty for N+ sessions, skip the verbose check-in and spend the freed time on self-directed work — design exploration, component research, pattern libraries. The standing orders still run, but the output shrinks.

## Application

- Recognize the idle pattern early (3+ sessions)
- After standing orders confirm "still empty," pivot to growth work
- Keep check-ins to one line: "Quill idle, standing orders done"
- Use the time gap for design sketches, research, or skill-building
