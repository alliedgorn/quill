# Design Process Lessons — Session 2

**Date**: 2026-03-20

## 1. Always poll before shipping design changes
Gorn values pack input. Even when designers converge on a solution, run it past the pack before implementing. The vote is part of the design process, not a rubber stamp.

## 2. Spec is my deliverable, not code in other repos
Quill delivers CSS specs and design direction. Implementation in oracle-v2 belongs to Dex (design) and Karo (code). Don't edit files in repos that aren't mine without asking first.

## 3. Don't rush past the conversation
Gorn's style: he wants the discussion, not just the result. The design lives in the back-and-forth. Slow down, present options, let people weigh in.

## 4. Coordinate vote formatting
When collaborating with another designer, agree on who posts the vote call to avoid duplicate/conflicting formats.
